# ypTrend 开发进展

更新时间：2026-04-16

## 当前阶段结论
- Web MVP 主链路已稳定可用，已进入“参数精调 + 体验优化 + Agent Skill 封装准备”阶段。
- 关键路径（多源采集 -> AI 判定 -> 统一门槛拦截 -> 飞书通知 -> SSE 实时推送 -> 前端实时展示）已闭环。

## 已完成（代码已落地）
1. 基础能力闭环
- 已完成监控规则管理、手动触发扫描、热点流/通知流展示。
- 已打通多源采集 + AI 判定 + 飞书通知 + SSE 实时推送。

2. 多源采集能力
- Twitter：`twitterapi.io`。
- Web：DuckDuckGo + Bing Web RSS + Bing News RSS 聚合。
- RSS：Google News RSS（多 locale）。

3. 可靠性与信号质量策略
- Twitter 严格原创过滤：reply/quote/retweet 不入候选。
- Twitter 互动分公式已落地：
  - `engagementScore = 1.5*likes + 2*retweets + 2*quotes + replies + views/100`
- Web/RSS 可比评分模型已落地：
  - `signalScore = sourceAuthority + recency + corroboration + contentQuality + aiConfidenceBonus`
- 全来源硬门槛已落地：
  - Twitter：按 `twitterQuality.notifyMinEngagementScore`（默认 2000）进行推送拦截。
  - Web/RSS：按 `nonTwitterSignal.highThreshold`（默认 70）进行高信号拦截。
- 7 天新鲜度窗口已全链路生效（采集/查询/通知/清理）。
- 低可信跨源佐证策略已生效（可配置最小独立来源数）。

4. 数据模型与接口增强
- `hot_items` 已新增并持久化以下结构化列：
  - `engagement_score`、`view_count`、`engagement_tier`
  - `signal_score`、`signal_tier`、`trust_level`
- 已增加数据库启动自动补列迁移逻辑，兼容旧库。
- `/api/hot-items` 返回结构化字段（不再依赖前端反解析 `raw_json`）。
- `/api/notifications` 与 SSE `notification` 事件已增强 AI 字段：
  - `ai_is_relevant`、`ai_is_credible`、`ai_confidence`、`ai_reason`、`ai_model` 等。

5. 前端交互与展示
- 热点流已改为“热点等级驱动”展示（高/中/低差异化 UI），去除此前不满意的冗余评分展示。
- 通知流已展示 AI 结论、可信/相关状态与置信度。
- 系统参数页已支持：
  - `X 推送阈值（互动分）`
  - `Web/RSS 高信号阈值`
