# ypTrend 需求说明

## 1. 项目目标
构建一个轻量、敏捷的热点监控工具，帮助用户第一时间发现并确认热点变化（重点关注 AI 大模型与 AI 编程方向），并自动通知。

## 2. 核心需求
1. 用户可手动输入监控关键词。
2. 系统自动从多信息源采集候选热点，避免单一来源。
3. 系统使用 AI 对候选内容进行真实性与相关性判断，降低假冒/蹭热度误报。
4. 命中后第一时间发送通知。
5. 系统定时搜集用户设定范围（如“AI 编程”）内的最新热点并展示。
6. 提供响应式 Web 页面（移动端和桌面端可用，视觉风格独特）。
7. 在 Web 版本跑通后，封装 Agent Skill 供其他 AI 使用。

## 3. 信息源要求（用户确认）
1. 网页搜索抓取（后端抓取搜索结果，不依赖付费搜索 API，控制抓取频率）。
2. Twitter(X) 数据：接入 twitterapi.io。
3. 至少再增加一个公共来源（如 RSS/新闻源）增强冗余。

## 4. 第三方服务要求（用户提供）
1. OpenRouter API Key：已提供。
2. twitterapi.io API Key：已提供。
3. 飞书 Webhook：已提供。
4. 飞书关键词校验：`trend`。

## 5. 通知要求
1. 通知渠道：飞书 Webhook。
2. 消息第一行必须为 `trend`（用于通过关键词校验）。
3. 通知内容包含：关键词、标题、来源、链接、AI 判断结论、置信度。

## 6. 非功能要求
1. 轻量化实现，优先可用性，减少过度工程化。
2. 支持基础去重、限频、失败重试。
3. 具备最小可用测试与验收脚本。

## 7. 里程碑顺序
1. 先完成 Web 版并验证功能可用。
2. 再封装 Agent Skill。

## 8. 新增需求变更（2026-04-15）
1. 当网页处于打开状态时，系统发现新热点后需要实时推送到浏览器。
2. 实时推送需直接更新前端热点流与通知流，无需等待手动刷新。
